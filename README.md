# URAN-4 Ionospheric Scintillation Analyzer

Программный комплекс для первичной обработки, фильтрации и частотно-временного анализа (спектроскопии) радиоастрономических сигналов радиотелескопа УРАН-4.

## Архитектура и алгоритмические решения

Программа построена по принципу модульного конвейера (pipeline). Все ключевые параметры вынесены в `core/config.py`, что позволяет проводить тонкую настройку алгоритмов без изменения логики ядра.

### 1. Очистка от индустриальных помех (Hampel Filter)
Используется для подавления импульсных помех в сыром сигнале.
* **Реализация:** Фильтр анализирует данные в скользящем окне `DEFAULT_WINDOW_SIZE` (15 отсчетов). Точки, отклоняющиеся от локальной медианы более чем на `DEFAULT_N_SIGMAS` (3.0 сигмы), заменяются медианным значением.

### 2. Сглаживание цифрового шума (Savitzky-Golay)
После удаления выбросов применяется фильтр Савицкого-Голея (`SAVGOL_POLYORDER = 2`). Это позволяет восстановить непрерывность сигнала, сохраняя при этом физическую амплитуду ионосферных мерцаний.

### 3. Выделение масштабов (Bandpass Filtering)
Сигнал проходит через полосовой фильтр Баттерворта 4-го порядка (`bandpass_filter`), параметры которого (граничные частоты) динамически меняются в зависимости от выбранного режима анализа: "Малые пузырьки" или "Большие облака".

### 4. Апсемплинг и интерполяция
Вместо прореживания применяется апсемплинг (x3) с использованием интерполяции PCHIP. Это позволяет избежать выбросов на резких скачках сигнала и повысить детализацию без потери частотного разрешения.

### 5. Частотно-временной анализ (CWT + SWT)
Используется непрерывное вейвлет-преобразование (CWT) с вейвлетом Морзе в комбинации с Синхросквизингом (SWT) на секундном разрешении (без прореживания). Это обеспечивает предельно узкие спектральные линии мерцаний.

### 6. Спектрально-корреляционный анализ
Для детального анализа Внезапного Стратосферного Потепления и скоростей облаков применяются:
* **Глобальный спектральный анализ:** Добавлена возможность спектрального анализа всего PM6 файла (кнопка в главном меню), что позволяет выявить долгопериодические 8-дневные осцилляции (150-600 с).
* **Multitaper PSD & F-Test:** Поиск детерминированных гармоник и основного периода осцилляций с помощью DPSS окон (7 тейперов). На графиках F-Test отображаются линии основного периода $T_0$ и его гармоник $2T, 3T$, а также используется 99% доверительный интервал.
* **Кросс-спектральный анализ:** Вычисление фазы и когерентности кросс-спектров интерферометрических каналов (20 МГц vs 25 МГц).
* **Оценка скоростей:** Расчет временных задержек и горизонтальных скоростей (модель: прохождение на просвет, разнесение лучей 2500 м). Анализ проводится с выделением Топ-5 доминирующих периодов.
* **Интерактивный UI:** Отдельная вкладка со встроенными саб-табами и сеткой графиков 2x2, точно повторяющая структуру эталонных результатов в MATLAB (включая цвета, сетку и отображение гармоник).

**Оптимизации производительности:**
* **Окно Тьюки:** Для подавления краевых эффектов ("вееров" на границах файла) перед анализом используется окно Тьюки с параметром `TUKEY_ALPHA = 0.1`, плавно сводящее края сигнала к нулю.

## Таблица основных параметров конфигурации (`core/config.py`)

| Параметр | Значение | Описание |
| :--- | :--- | :--- |
| `DEFAULT_WINDOW_SIZE` | 15 | Размер окна фильтра Хампеля |
| `DEFAULT_N_SIGMAS` | 3.0 | Порог отклонения для удаления спайков |
| `SAVGOL_POLYORDER` | 2 | Порядок полинома Савицкого-Голея |
| `TUKEY_ALPHA` | 0.1 | Степень сглаживания краев сигнала |
| `MTM_N_TAPERS` | 7 | Количество DPSS тейперов для метода Multitaper |
| `MTM_NW` | 4.0 | Произведение время-полоса для окон DPSS |
| `FTEST_CONFIDENCE` | 0.99 | Уровень значимости для критерия Фишера (F-Test), F=6.93 |
| `CROSS_SPECTRUM_DX` | 2500 | Расстояние между лучами телескопа (метры) |
| `VELOCITY_N_PEAKS` | 3 | Количество пиков для оценки скорости |

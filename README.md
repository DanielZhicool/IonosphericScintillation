# URAN-4 Ionospheric Scintillation Analyzer

Программный комплекс для первичной обработки, фильтрации и частотно-временного анализа (спектроскопии) радиоастрономических сигналов радиотелескопа УРАН-4.

## Архитектура и алгоритмические решения

Программа построена по принципу модульного конвейера (pipeline). Все ключевые параметры вынесены в централизованный конфигурационный файл `core/config.py`, что обеспечивает гибкость настройки без изменения логики ядра.

### 1. Конфигурация и модульность
Проект переведен на модульную систему параметров. Все константы обработки — от размеров окон фильтрации до коэффициентов сжатия — управляются через файл `core/config.py`. Это позволяет быстро адаптировать алгоритмы под различные условия наблюдений, не затрагивая реализацию самих функций.

### 2. Хронологический анализ и кластеризация сессий
Сырые логи наблюдений (`regi`) и файлы данных (`PM6`) имеют асинхронную структуру.
* **Решение:** Программа кластеризует данные на основе порога `GAP_THRESHOLD = 3600` секунд. Это обеспечивает физическую непрерывность фазы внутри каждого сеанса, исключая артефакты Гиббса, возникающие при искусственном склеивании разрозненных временных интервалов.

### 3. Очистка от индустриальных помех (Hampel Filter)
Для подавления импульсных помех используется фильтр Хампеля.
* **Реализация:** Функция `clean_and_smooth_signal` вычисляет локальную медиану и абсолютное медианное отклонение (MAD), масштабированное с коэффициентом 1.4826. Точки, отклоняющиеся более чем на `DEFAULT_N_SIGMAS`, заменяются локальной медианой, что позволяет эффективно удалять спайки, сохраняя структуру полезного сигнала.

### 4. Сглаживание (Savitzky-Golay)
* **Реализация:** После удаления выбросов применяется фильтр Савицкого-Голея с полиномом степени `SAVGOL_POLYORDER`. Это позволяет устранить "ступенчатость" сигнала, возникшую после работы медианного фильтра, сохраняя амплитудную целостность мерцаний.

### 5. Выделение масштабов (Bandpass Filtering)
Сигнал проходит через полосовой фильтр Баттерворта 4-го порядка (`bandpass_filter`), параметры которого (граничные частоты) динамически меняются в зависимости от выбранного режима анализа: "Малые пузырьки" или "Большие облака".

### 6. Частотно-временной анализ (FSST)
Для анализа используется синхро-сжатое преобразование Фурье (`ssq_stft`), обеспечивающее высокую спектральную локализацию.

**Оптимизации производительности:**
* **Децимация:** Для режима "Большие облака" применяется прореживание сигнала с коэффициентом `DECIMATION_Q`, что ускоряет расчеты в десятки раз без потери точности для ультранизких частот.
* **Окно Тьюки:** Внедрена оконная функция Тьюки с коэффициентом `TUKEY_ALPHA`, которая плавно сводит края сигнала к нулю. Это полностью устраняет "веерные" краевые артефакты, вызванные переходными процессами фильтров.
* **Защита памяти:** Алгоритм автоматически ограничивает размер FFT через `NFFT_CAP` и использует адаптивный zero-padding, предотвращая переполнение оперативной памяти (OOM) при обработке длинных временных рядов.

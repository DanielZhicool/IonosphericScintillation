# URAN-4 Ionospheric Scintillation Analyzer

Программный комплекс для первичной обработки, фильтрации и частотно-временного анализа (спектроскопии) радиоастрономических сигналов радиотелескопа УРАН-4.

## Архитектура и алгоритмические решения

Программа построена по принципу модульного конвейера (pipeline). Все ключевые параметры вынесены в `core/config.py`, что позволяет проводить тонкую настройку алгоритмов без изменения логики ядра.

### 1. Очистка от индустриальных помех (Hampel Filter)
Используется для подавления импульсных помех в сыром сигнале.
* **Реализация:** Фильтр анализирует данные в скользящем окне `DEFAULT_WINDOW_SIZE` (15 отсчетов). Точки, отклоняющиеся от локальной медианы более чем на `DEFAULT_N_SIGMAS` (3.0 сигмы), заменяются медианным значением.

### 2. Сглаживание цифрового шума (Savitzky-Golay)
После удаления выбросов применяется фильтр Савицкого-Голея (`SAVGOL_POLYORDER = 2`). Это позволяет восстановить непрерывность сигнала, сохраняя при этом физическую амплитуду ионосферных мерцаний.

### 3. Частотно-временной анализ (FSST)
Метод синхро-сжатого преобразования Фурье обеспечивает высокую локализацию спектральных линий.

**Оптимизации производительности:**
* **Децимация:** Для режима "Большие облака" применяется прореживание сигнала с коэффициентом `DECIMATION_Q = 10`. Это ускоряет расчеты в десятки раз, позволяя анализировать ультранизкие частоты (ниже `LOW_FREQ_HIGHCUT = 0.01` Гц) без перегрузки CPU.
* **Окно Тьюки:** Для подавления краевых эффектов ("вееров" на границах файла) используется окно Тьюки с параметром `TUKEY_ALPHA = 0.1`, плавно сводящее края сигнала к нулю.
* **Управление памятью:** Размер FFT ограничен константой `NFFT_CAP = 32768` для предотвращения переполнения оперативной памяти (OOM) при работе с длинными временными рядами.

## Таблица основных параметров конфигурации (`core/config.py`)

| Параметр | Значение | Описание |
| :--- | :--- | :--- |
| `DEFAULT_WINDOW_SIZE` | 15 | Размер окна фильтра Хампеля |
| `DEFAULT_N_SIGMAS` | 3.0 | Порог отклонения для удаления спайков |
| `SAVGOL_POLYORDER` | 2 | Порядок полинома Савицкого-Голея |
| `DECIMATION_Q` | 10 | Коэффициент сжатия сигнала |
| `LOW_FREQ_HIGHCUT` | 0.01 | Порог частоты для включения оптимизаций |
| `TUKEY_ALPHA` | 0.1 | Степень сглаживания краев сигнала |
| `NFFT_CAP` | 32768 | Максимальный размер окна FFT |
